# Components Site Template

A TanStack Start + React template for documenting UI component libraries. Write MDX docs with live previews, generate a [shadcn](https://ui.shadcn.com/) registry, and let users install your components with `shadcn add`.

## Stack

- [TanStack Start](https://tanstack.com/start) and TanStack Router for the app shell and routing
- [fumadocs-mdx](https://fumadocs.dev/) and fumadocs-core for MDX parsing and content loading
- React docs primitives for previews, source tabs, code blocks, install commands, and props tables
- Tailwind CSS v4 with shadcn-style CSS variables
- [shadcn CLI](https://ui.shadcn.com/docs/cli) registry generation via `registry.json`
- Cloudflare Workers deployment via Wrangler

## Getting Started

### Use this template

1. Click **Use this template** on GitHub, or clone the repository.
2. Install dependencies:

```bash
pnpm install
```

3. Start the dev server:

```bash
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000). Docs are served at `/docs`.

`npm` and `bun` work too — replace `pnpm` with your package manager in the commands below.

### Customize for your library

Before publishing, update these to match your project:

| File                            | What to change                                                     |
| ------------------------------- | ------------------------------------------------------------------ |
| `registry.json`                 | `name` and `homepage` (your production URL)                        |
| `package.json`                  | `name`                                                             |
| `wrangler.jsonc`                | `name` (Cloudflare Worker name)                                    |
| `content/docs/components/*.mdx` | Install command URLs (`http://localhost:3000/r/...` → your domain) |
| `src/lib/theme.ts`              | Local storage keys for theme/UI preferences                        |
| `src/routes/index.tsx`          | Landing page copy and links                                        |

After changing `registry.json`, rebuild the registry output:

```bash
pnpm registry:build
```

## Project Structure

```
content/docs/          MDX documentation pages
registry/default/      Installable component source
  ui/                  Component implementations
  examples/            Demo components used in docs
  lib/                 Shared utilities (e.g. cn)
  styles/              Theme CSS for the registry
registry.json          shadcn registry manifest
public/r/              Generated registry JSON (do not edit)
src/                   App shell, docs primitives, routes
source.config.ts       fumadocs-mdx configuration
```

Path aliases:

- `@/*` and `~/*` → `./src/*`
- `registry/*` → `./registry/*`
- `collections/*` → `./.source/*` (generated by fumadocs-mdx)

## Adding a Component

1. Create the component in `registry/default/ui/<name>.tsx`.
2. Create a demo in `registry/default/examples/<name>-demo.tsx`.
3. Add an entry to `registry.json` (include `registryDependencies` like `utils` and `theme` for UI components).
4. Run `pnpm registry:build` to write JSON files to `public/r/`.
5. Write a docs page at `content/docs/components/<name>.mdx`.

To add the page to the sidebar, place it under `content/docs/components/`. Folder structure and `meta.json` files control navigation.

### Example docs page

```mdx
---
title: Button
description: A compact action primitive.
component: Button
registry: button
---

import { ButtonDemo } from "registry/default/examples/button-demo";
import buttonDemoSource from "registry/default/examples/button-demo.tsx?raw";
import buttonSource from "registry/default/ui/button.tsx?raw";
```

### Available MDX components

These are registered in `src/components/mdx/mdx-components.tsx`:

- `ComponentTabs` — tabbed preview and source layout
- `ComponentPreview` — live demo with optional `name` for the tab label
- `ComponentSource` — syntax-highlighted source (`code`, `language`, `title`)
- `CodeSnippet` / `CodeSnippet.Command` — install commands with package-manager tabs

Import registry demos and source with Vite `?raw` imports so docs always match the installable files.

## Registry

`pnpm registry:build` runs `shadcn build` and outputs item JSON to `public/r/`. Each component gets a file like `public/r/button.json`.

Users install components into their own projects:

```bash
npx shadcn@latest add https://your-site.com/r/button.json
```

The registry includes `utils` and `theme` items. UI components should declare `"registryDependencies": ["utils", "theme"]` so consumers get the required styles and helpers.

`public/r/` is generated — edit `registry.json` and source files instead, then rebuild.

## Styling

- Theme tokens live in `src/styles/app.css` using Tailwind v4 `@theme inline`.
- shadcn CSS variables (`--background`, `--foreground`, `--primary`, etc.) are the source of truth.
- Dark mode uses `data-theme="dark"` on `<html>`.
- Registry theme CSS is in `registry/default/styles/style.css`.

`components.json` follows the shadcn **new-york** style with CSS variables enabled.

## Deployment

The template is configured for [Cloudflare Workers](https://developers.cloudflare.com/workers/) via `@cloudflare/vite-plugin` and Wrangler.

1. Update `registry.json` `homepage` and MDX install URLs to your production domain.
2. Run `pnpm registry:build` so `public/r/` is up to date.
3. Deploy:

```bash
pnpm deploy
```

You can also deploy the `dist` output from `pnpm build` to any platform that supports TanStack Start.

## Generated Files

Do not edit these directly:

- `.source/` — fumadocs-mdx generated types (run `pnpm typegen` to regenerate)
- `src/routeTree.gen.ts` — TanStack Router route tree
- `public/r/` — shadcn registry output (run `pnpm registry:build`)
